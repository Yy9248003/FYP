## 幻灯片 3: 项目背景与问题陈述

### 传统考试系统的痛点
- ❌ **评分效率低**: 主观题需要人工批改，耗时耗力（2-5分钟/题）
- ❌ **出题工作量大**: 教师需要手动创建大量题目（10分钟/题）
- ❌ **缺乏智能分析**: 无法深入分析学生错误原因
- ❌ **学习反馈不足**: 学生难以获得个性化学习建议
- ❌ **管理复杂**: 多角色权限管理繁琐
- ❌ **部署困难**: 传统系统部署复杂，需要专业运维

### 市场机会

## 幻灯片 5: 技术架构 - 整体架构

```
┌─────────────────────────────────────────────────┐
│              前端层 (Frontend)                    │
│  Vue.js 3.x + View UI Plus + Vue Router + Vuex  │
└──────────────────┬──────────────────────────────┘
                   │ HTTP/REST API
┌──────────────────▼──────────────────────────────┐
│              后端层 (Backend)                     │
│  Django 4.1.3 + Django REST Framework            │
│  + JWT认证 + Redis缓存                            │
└──────────────────┬──────────────────────────────┘
                   │
┌──────────────────▼──────────────────────────────┐
│              数据层 (Database)                    │
│  MySQL 8.0+ (关系型数据库)                        │
└──────────────────┬──────────────────────────────┘
                   │
┌──────────────────▼──────────────────────────────┐
│              AI服务层 (AI Service)                │
│  智谱AI (GLM-4-Flash) - 智能评分与出题            │
└─────────────────────────────────────────────────┘
```

- **前后端分离**: 清晰的职责划分，易于维护

  - 选择题：精确匹配
  - 填空题：语义理解评分
  - 判断题：逻辑判断
  - 编程题：代码逻辑分析

- 📊 **智能分析**:
  - 详细评分反馈
  - 错误原因分析
  - 知识点掌握评估
  - 改进建议

- 🔄 **降级机制**:
  - AI评分失败时自动使用传统评分
  - 确保系统稳定性

### 评分示例
```
题目: "解释什么是面向对象编程"
学生答案: "面向对象是一种编程思想..."
AI评分: 8.5/10
反馈: "答案基本正确，但缺少封装、继承、多态等核心概念的具体说明"
```

- ✅ 降级机制保证稳定性
- **结果**: 成功集成，稳定性99%+

### 挑战2: Docker部署
**问题**: 容器间通信和依赖管理  
**解决方案**:
- ✅ Docker Compose多容器编排
- ✅ 健康检查机制确保服务可用
- ✅ 环境变量统一管理
- ✅ 数据持久化配置
- **结果**: 一键部署，零配置启动

### 挑战3: 权限管理
**问题**: 多角色权限控制复杂  
**解决方案**:
- ✅ 基于角色的访问控制(RBAC)

- ✅ 后端API权限中间件验证
- ✅ 前后端双重权限检查
- **结果**: 权限控制精确，无权限泄露

### 挑战4: AI评分准确性
**问题**: 确保AI评分结果可靠

- ✅ 针对不同题型优化提示词

**Q1: 为什么选择智谱AI而不是OpenAI?**

A: 我们实现了降级机制，AI评分失败时自动使用传统评分。同时通过大量测试优化提示词。

A: 使用JWT认证、权限控制、SQL注入防护、XSS防护等多层安全措施。
