### 2. N+1查询问题修复 ✅

#### 优化的查询方法

1. **PractisesView.getPageInfos**
   - 优化前: 1 + 2N 个查询
   - 优化后: 2 个查询

2. **TasksView.getStudentTasks**
   - 优化前: 1 + 3N 个查询
   - 优化后: 2 个查询
   - 技术: `select_related('project', 'teacher')` + 批量查询任务日志

3. **ExamLogsView.getInfo**
   - 优化前: 1 + 2N 个查询
   - 优化后: 2 个查询
   - 技术: `select_related` + `prefetch_related` + 批量查询选项

4. **ExamLogsView.getPageStudentLogs**
   - 优化前: 1 + 3N 个查询

5. **ExamLogsView.getPageTeacherLogs**
   - 优化前: 1 + 3N 个查询

## 🔧 技术实现细节

### 1. select_related 使用

```python
# 优化前
tasks = models.Tasks.objects.filter(grade=student.grade)
for task in tasks:
    print(task.project.name)  # N+1查询

# 优化后
tasks = models.Tasks.objects.filter(grade=student.grade).select_related('project', 'teacher')
for task in tasks:
    print(task.project.name)  # 无额外查询
```

### 2. 批量查询优化

```python
# 优化前
for item in items:
    count = models.Options.objects.filter(practise_id=item.id).count()  # N+1查询

# 优化后
practise_ids = [item.id for item in items]
option_counts = models.Options.objects.filter(
    practise_id__in=practise_ids
).values('practise_id').annotate(count=Count('id'))
counts_dict = {item['practise_id']: item['count'] for item in option_counts}
```
