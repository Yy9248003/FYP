## 🎯 部署方案概览

### 方案对比

| 方案 | 优点 | 缺点 | 适用场景 | 成本 |
|------|------|------|----------|------|
| **云服务器 + Docker** | 灵活、可控、成本低 | 需要运维 | 中小型项目 | 低-中 |
| **云容器服务** | 自动化、易扩展 | 成本较高 | 中大型项目 | 中-高 |
| **Serverless** | 按需付费、零运维 | 冷启动、限制多 | 小型项目 | 低 |

## 🚀 方案一：云服务器 + Docker（推荐）

### 适用平台
- ✅ 阿里云 ECS
- ✅ 腾讯云 CVM
- ✅ 华为云 ECS
- ✅ AWS EC2
- ✅ Azure VM

### 前置准备

1. **购买云服务器**
   - 推荐配置：2核4G，带宽 5Mbps
   - 系统：Ubuntu 20.04/22.04 或 CentOS 7/8
   - 开放端口：80, 443, 8000, 8080, 3306

2. **准备域名**（可选）
   - 购买域名并完成备案（国内服务器需要）

### 部署步骤

#### 1. 连接服务器

```bash
# SSH 连接服务器
ssh root@your-server-ip
```

#### 2. 安装 Docker 和 Docker Compose

```bash
# Ubuntu/Debian
curl -fsSL https://get.docker.com -o get-docker.sh
sh get-docker.sh

# 安装 Docker Compose
sudo curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose

# 验证安装
docker --version
docker-compose --version
```

#### 3. 上传项目代码

**方法 A：使用 Git（推荐）**

```bash
# 在服务器上克隆项目
cd /opt
git clone https://github.com/your-username/25FYP.git
cd 25FYP
```

**方法 B：使用 SCP**

```bash
# 在本地执行
scp -r /path/to/25FYP root@your-server-ip:/opt/
```

#### 4. 配置环境变量

```bash
# 进入项目目录
cd /opt/25FYP

# 创建生产环境配置文件
cat > .env.production << EOF
# 数据库配置
DB_HOST=your-rds-host  # 如果使用云数据库
DB_PORT=3306
DB_NAME=db_exam
DB_USER=examuser
DB_PASSWORD=your-strong-password

# Django 配置
SECRET_KEY=your-secret-key-here
DEBUG=False
ALLOWED_HOSTS=your-domain.com,www.your-domain.com,your-server-ip

# CORS 配置
CORS_ALLOWED_ORIGINS=https://your-domain.com,https://www.your-domain.com

# AI 配置
ZHIPUAI_API_KEY=your-api-key
ZHIPUAI_MODEL=glm-4-flash
EOF
```

#### 5. 修改 docker-compose.yml

```yaml
version: '3.8'

services:
  db:
    image: mysql:8.0
    container_name: fyp_mysql
    restart: always
    environment:
      MYSQL_ROOT_PASSWORD: ${DB_ROOT_PASSWORD:-your-strong-password}
      MYSQL_DATABASE: ${DB_NAME:-db_exam}
      MYSQL_USER: ${DB_USER:-examuser}
      MYSQL_PASSWORD: ${DB_PASSWORD:-your-strong-password}
    volumes:
      - mysql_data:/var/lib/mysql
      - ./database/db_exam.sql:/docker-entrypoint-initdb.d/01-structure.sql
    ports:
      - "127.0.0.1:3306:3306"  # 只允许本地访问
    networks:
      - fyp_network

  backend:
    build:
      context: ./source/server
      dockerfile: ../../docker/backend/Dockerfile
    container_name: fyp_backend
    restart: always
    depends_on:
      - db
    command: >
      bash -c "
      python manage.py migrate --noinput &&
      python manage.py collectstatic --noinput &&
      gunicorn server.wsgi:application --bind 0.0.0.0:8000 --workers 4
      "
    environment:
      - DB_HOST=db
      - DB_PORT=3306
      - DB_NAME=${DB_NAME:-db_exam}
      - DB_USER=${DB_USER:-examuser}
      - DB_PASSWORD=${DB_PASSWORD:-your-strong-password}
      - SECRET_KEY=${SECRET_KEY}
      - DEBUG=False
      - ALLOWED_HOSTS=${ALLOWED_HOSTS}
    volumes:
      - ./source/server:/app
      - static_files:/app/staticfiles
    networks:
      - fyp_network

  frontend:
    build:
      context: ./source/client
      dockerfile: ../../docker/frontend/Dockerfile.prod
    container_name: fyp_frontend
    restart: always
    depends_on:
      - backend
    environment:
      - VUE_APP_API_BASE_URL=https://api.your-domain.com
    ports:
      - "80:80"
    networks:
      - fyp_network

volumes:
  mysql_data:
  static_files:

networks:
  fyp_network:
    driver: bridge
```

#### 6. 安装 Gunicorn（生产环境 WSGI 服务器）

修改 `source/server/requirements.txt`，添加：

```txt
gunicorn==21.2.0
```

#### 7. 启动服务

```bash
# 构建并启动
docker-compose up -d --build

# 查看日志
docker-compose logs -f

# 查看服务状态
docker-compose ps
```

#### 8. 配置 Nginx 反向代理（可选但推荐）

```bash
# 安装 Nginx
sudo apt-get update
sudo apt-get install -y nginx

# 创建 Nginx 配置
sudo nano /etc/nginx/sites-available/fyp
```

Nginx 配置内容：

```nginx
server {
    listen 80;
    server_name your-domain.com www.your-domain.com;

    # 前端静态文件
    location / {
        proxy_pass http://localhost:80;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 后端 API
    location /api/ {
        proxy_pass http://localhost:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 静态文件
    location /static/ {
        alias /var/www/static/;
        expires 30d;
        add_header Cache-Control "public, immutable";
    }
}
```

```bash
# 启用配置
sudo ln -s /etc/nginx/sites-available/fyp /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

## 🐳 方案二：云容器服务

### 阿里云容器服务 ACK

#### 1. 创建 Kubernetes 集群

1. 登录阿里云控制台
2. 进入「容器服务 Kubernetes 版」
3. 创建集群（选择托管版，2核4G节点）

#### 2. 推送镜像到容器镜像服务

```bash
# 登录阿里云容器镜像服务
docker login --username=your-username registry.cn-hangzhou.aliyuncs.com

# 构建并推送后端镜像
cd source/server
docker build -t registry.cn-hangzhou.aliyuncs.com/your-namespace/fyp-backend:latest .
docker push registry.cn-hangzhou.aliyuncs.com/your-namespace/fyp-backend:latest

# 构建并推送前端镜像
cd ../client
docker build -f ../../docker/frontend/Dockerfile.prod -t registry.cn-hangzhou.aliyuncs.com/your-namespace/fyp-frontend:latest .
docker push registry.cn-hangzhou.aliyuncs.com/your-namespace/fyp-frontend:latest
```

#### 3. 创建 Kubernetes 部署文件

创建 `k8s/deployment.yaml`:

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: fyp-backend
spec:
  replicas: 2
  selector:
    matchLabels:
      app: fyp-backend
  template:
    metadata:
      labels:
        app: fyp-backend
    spec:
      containers:
      - name: backend
        image: registry.cn-hangzhou.aliyuncs.com/your-namespace/fyp-backend:latest
        ports:
        - containerPort: 8000
        env:
        - name: DB_HOST
          value: "your-rds-endpoint"
        - name: DB_NAME
          value: "db_exam"
        - name: SECRET_KEY
          valueFrom:
            secretKeyRef:
              name: fyp-secrets
              key: secret-key
---
apiVersion: v1
kind: Service
metadata:
  name: fyp-backend-service
spec:
  selector:
    app: fyp-backend
  ports:
  - port: 80
    targetPort: 8000
  type: LoadBalancer
```

### 腾讯云容器服务 TKE

步骤类似，使用腾讯云容器镜像服务 TCR。

## 🔒 域名和 HTTPS 配置

### 使用 Let's Encrypt 免费 SSL 证书

```bash
# 安装 Certbot
sudo apt-get update
sudo apt-get install -y certbot python3-certbot-nginx

# 获取证书
sudo certbot --nginx -d your-domain.com -d www.your-domain.com

# 自动续期
sudo certbot renew --dry-run
```

### 修改 Nginx 配置支持 HTTPS

```nginx
server {
    listen 443 ssl http2;
    server_name your-domain.com www.your-domain.com;

    ssl_certificate /etc/letsencrypt/live/your-domain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/your-domain.com/privkey.pem;

    # SSL 配置
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;

    location / {
        proxy_pass http://localhost:80;
        # ... 其他配置
    }

    location /api/ {
        proxy_pass http://localhost:8000;
        # ... 其他配置
    }
}

# HTTP 重定向到 HTTPS
server {
    listen 80;
    server_name your-domain.com www.your-domain.com;
    return 301 https://$server_name$request_uri;
}
```

## 🗄️ 数据库迁移到云数据库

### 使用阿里云 RDS MySQL

#### 1. 创建 RDS 实例

1. 登录阿里云控制台
2. 进入「云数据库 RDS MySQL」
3. 创建实例（推荐：高可用版，2核4G）

#### 2. 配置白名单

将服务器 IP 添加到 RDS 白名单

#### 3. 导入数据

```bash
# 在服务器上执行
mysql -h your-rds-endpoint.mysql.rds.aliyuncs.com \
      -u examuser -p \
      db_exam < database/db_exam.sql
```

#### 4. 修改 docker-compose.yml

```yaml
backend:
  environment:
    DB_HOST: your-rds-endpoint.mysql.rds.aliyuncs.com
    DB_PORT: 3306
    # ... 其他配置
```

#### 5. 移除本地数据库服务

从 `docker-compose.yml` 中删除 `db` 服务

## ❓ 常见问题

### Q1: 如何备份数据？

```bash
# 备份数据库
docker-compose exec db mysqldump -u examuser -p db_exam > backup_$(date +%Y%m%d).sql

# 恢复数据库
docker-compose exec -T db mysql -u examuser -p db_exam < backup_20231227.sql
```

### Q2: 如何更新代码？

```bash
# 拉取最新代码
git pull

# 重新构建并启动
docker-compose up -d --build

# 执行数据库迁移
docker-compose exec backend python manage.py migrate
```

### Q3: 如何查看日志？

```bash
# 查看所有服务日志
docker-compose logs -f

# 查看特定服务日志
docker-compose logs -f backend
docker-compose logs -f frontend
```

### Q4: 如何扩容？

```bash
# 增加后端实例
docker-compose up -d --scale backend=3
```

### Q5: 如何配置自动重启？

在 `docker-compose.yml` 中已配置 `restart: always`

## 🎯 推荐配置

### 小型项目（< 100 用户）
- 服务器：1核2G，带宽 3Mbps
- 数据库：本地 MySQL 或云数据库基础版
- 成本：约 ¥50-100/月

### 中型项目（100-1000 用户）
- 服务器：2核4G，带宽 5Mbps
- 数据库：云数据库高可用版
- 成本：约 ¥200-500/月

### 大型项目（> 1000 用户）
- 服务器：4核8G+，带宽 10Mbps+
- 数据库：云数据库高可用版 + 读写分离
- 使用容器服务或 Kubernetes
- 成本：约 ¥1000+/月
