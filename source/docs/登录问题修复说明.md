# 登录问题修复说明

**修复时间**: 2025-11-16  
**问题**: 登录时出现网络错误（Network Error）

---

## 🔍 问题分析

### 错误信息
- **前端错误**: `AxiosError: Network Error`
- **后端错误**: `ModuleNotFoundError: No module named 'app.comm'`

### 根本原因

1. **后端中间件导入错误**
   - 在 `settings.py` 中添加了查询监控中间件
   - 中间件路径配置有问题，导致Django无法启动

2. **前端API地址配置**
   - `http.js` 中硬编码了 `localhost:8000`
   - 在Docker环境中可能需要使用环境变量

---

## ✅ 修复方案

### 1. 修复后端中间件问题

**文件**: `source/server/server/settings.py`

**修改前**:
```python
MIDDLEWARE = [
    # ...
    'app.comm.query_monitor.QueryMonitorMiddleware' if DEBUG else None,
]
MIDDLEWARE = [m for m in MIDDLEWARE if m is not None]
```

**修改后**:
```python
MIDDLEWARE = [
    # ...
]

# 查询监控中间件（暂时注释掉，避免导入错误）
# if DEBUG:
#     try:
#         from app.comm.query_monitor import QueryMonitorMiddleware
#         MIDDLEWARE.append('app.comm.query_monitor.QueryMonitorMiddleware')
#     except ImportError:
#         pass
```

**说明**: 暂时注释掉查询监控中间件，避免导入错误影响系统启动。

### 2. 修复前端API地址配置

**文件**: `source/client/src/utils/http.js`

**修改前**:
```javascript
const service = axios.create({
	baseURL: 'http://localhost:8000/api',
	timeout: 15000 
})
```

**修改后**:
```javascript
// 从环境变量获取API地址，如果没有则使用默认值
const getBaseURL = () => {
	// 优先使用环境变量（Docker环境）
	if (process.env.VUE_APP_API_BASE_URL) {
		return `${process.env.VUE_APP_API_BASE_URL}/api`
	}
	// 开发环境默认值
	return 'http://localhost:8000/api'
}

const service = axios.create({
	baseURL: getBaseURL(),
	timeout: 15000 
})
```

**说明**: 支持从环境变量读取API地址，兼容Docker和本地开发环境。

- **Docker环境**: 使用环境变量 `VUE_APP_API_BASE_URL`
