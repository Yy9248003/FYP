# ☁️ 云端部署快速参考

## 🚀 快速开始（5分钟部署）

### 前提条件
- ✅ 已购买云服务器（推荐：2核4G，Ubuntu 20.04+）
- ✅ 已安装 Docker 和 Docker Compose
- ✅ 已开放端口：80, 443, 8000

### 一键部署

```bash
# 1. 连接服务器
ssh root@your-server-ip

# 2. 克隆项目
cd /opt
git clone https://github.com/your-username/25FYP.git
cd 25FYP

# 3. 运行部署脚本
chmod +x deploy/cloud-deploy.sh
./deploy/cloud-deploy.sh

# 4. 访问系统
# 前端: http://your-server-ip
# 后端: http://your-server-ip:8000
```

## 📋 手动部署步骤

### 1. 安装 Docker

```bash
curl -fsSL https://get.docker.com -o get-docker.sh
sh get-docker.sh
sudo curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose
```

### 2. 配置环境变量

```bash
# 创建 .env 文件
cat > .env << EOF
DB_PASSWORD=your-strong-password
SECRET_KEY=$(python3 -c 'import secrets; print(secrets.token_urlsafe(50))')
ALLOWED_HOSTS=your-domain.com,your-server-ip
DEBUG=False
EOF
```

### 3. 启动服务

```bash
# 使用生产配置
docker-compose -f docker-compose.prod.yml up -d --build

# 查看日志
docker-compose -f docker-compose.prod.yml logs -f
```

## 🌐 配置域名和 HTTPS

### 1. 安装 Nginx

```bash
sudo apt-get update
sudo apt-get install -y nginx certbot python3-certbot-nginx
```

### 2. 配置 Nginx

```bash
sudo nano /etc/nginx/sites-available/fyp
```

```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://localhost:80;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    location /api/ {
        proxy_pass http://localhost:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

### 3. 启用配置并获取 SSL 证书

```bash
sudo ln -s /etc/nginx/sites-available/fyp /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
sudo certbot --nginx -d your-domain.com
```

## 🗄️ 使用云数据库（推荐）

### 阿里云 RDS

1. 创建 RDS MySQL 实例
2. 配置白名单（添加服务器 IP）
3. 修改 `.env`:

```bash
DB_HOST=your-rds-endpoint.mysql.rds.aliyuncs.com
DB_PORT=3306
DB_NAME=db_exam
DB_USER=examuser
DB_PASSWORD=your-password
```

4. 导入数据：

```bash
mysql -h your-rds-endpoint -u examuser -p db_exam < database/db_exam.sql
```

5. 从 `docker-compose.prod.yml` 删除 `db` 服务

## 📊 监控和日志

### 查看实时日志

```bash
# 所有服务
docker-compose logs -f

# 特定服务
docker-compose logs -f backend
```

### 查看资源使用

```bash
docker stats
```

### 设置日志轮转

在 `docker-compose.prod.yml` 中添加：

```yaml
backend:
  logging:
    driver: "json-file"
    options:
      max-size: "10m"
      max-file: "3"
```

## 🔒 安全建议

1. **修改默认密码**
   ```bash
   # 修改数据库密码
   DB_PASSWORD=your-strong-password-here
   ```

2. **设置防火墙**
   ```bash
   sudo ufw allow 22/tcp
   sudo ufw allow 80/tcp
   sudo ufw allow 443/tcp
   sudo ufw enable
   ```

3. **定期备份**
   ```bash
   # 创建备份脚本
   cat > /opt/backup.sh << 'EOF'
   #!/bin/bash
   DATE=$(date +%Y%m%d_%H%M%S)
   docker-compose exec -T db mysqldump -u examuser -p$DB_PASSWORD db_exam > /opt/backups/backup_$DATE.sql
   # 保留最近7天的备份
   find /opt/backups -name "backup_*.sql" -mtime +7 -delete
   EOF
   
   # 添加到 crontab（每天凌晨2点备份）
   echo "0 2 * * * /opt/backup.sh" | crontab -
   ```

4. **更新系统**
   ```bash
   sudo apt-get update
   sudo apt-get upgrade -y
   ```

## 🐛 故障排除

### 问题1: 服务无法启动

```bash
# 查看详细日志
docker-compose logs backend
docker-compose logs frontend

# 检查端口占用
netstat -tulpn | grep :8000
```

### 问题2: 数据库连接失败

```bash
# 测试数据库连接
docker-compose exec backend python manage.py dbshell

# 检查数据库服务
docker-compose ps db
docker-compose logs db
```

### 问题3: 静态文件404

```bash
# 重新收集静态文件
docker-compose exec backend python manage.py collectstatic --noinput
```

### 问题4: 内存不足

```bash
# 查看内存使用
free -h
docker stats

# 增加交换空间
sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
```
