# 🔧 环境配置文件说明

## 📋 概述

智能考试系统使用环境变量来管理配置，支持开发、测试和生产环境的不同配置需求。主要配置文件包括：

- `env.example` - 环境配置模板文件
- `.env` - 实际环境配置文件（需要用户创建）

---

## 📁 配置文件结构

### 1. env.example (配置模板)
这是环境配置的模板文件，包含了所有可配置的环境变量及其默认值。

### 2. .env (实际配置)
这是实际使用的环境配置文件，需要用户根据实际情况修改。

---

## 🔧 环境变量详细说明

### 数据库配置 (Database Configuration)

```env
# 数据库连接URL (推荐使用)
DATABASE_URL=mysql://root:123456@localhost:3306/db_exam

# 或者使用单独的数据库配置变量
DB_NAME=db_exam                    # 数据库名称
DB_USER=root                       # 数据库用户名
DB_PASSWORD=123456                 # 数据库密码
DB_HOST=localhost                  # 数据库主机地址
DB_PORT=3306                       # 数据库端口
```

**配置说明:**
- `DATABASE_URL`: 完整的数据库连接URL，格式为 `mysql://用户名:密码@主机:端口/数据库名`
- `DB_NAME`: 数据库名称，默认为 `db_exam`
- `DB_USER`: 数据库用户名，默认为 `root`
- `DB_PASSWORD`: 数据库密码，请修改为您的实际密码
- `DB_HOST`: 数据库主机地址，本地开发使用 `localhost`
- `DB_PORT`: 数据库端口，MySQL默认端口为 `3306`

### Django配置 (Django Configuration)

```env
# Django安全密钥 (必须修改)
SECRET_KEY=your-secret-key-here

# 调试模式 (开发环境设为True，生产环境设为False)
DEBUG=True

# 允许的主机 (生产环境需要修改)
ALLOWED_HOSTS=localhost,127.0.0.1,0.0.0.0
```

**配置说明:**
- `SECRET_KEY`: Django的安全密钥，用于加密和签名，**必须修改为随机字符串**
- `DEBUG`: 调试模式，开发时设为 `True`，生产环境必须设为 `False`
- `ALLOWED_HOSTS`: 允许访问的主机列表，生产环境需要添加实际域名

### AI服务配置 (AI Service Configuration)

```env
# 智谱AI API密钥
ZHIPUAI_API_KEY=your-zhipuai-api-key

# 智谱AI API基础URL
ZHIPUAI_BASE_URL=https://open.bigmodel.cn/api/paas/v4

# 使用的AI模型（免费模型：glm-4-flash）
ZHIPUAI_MODEL=glm-4-flash
```

**配置说明:**
- `ZHIPUAI_API_KEY`: 智谱AI的API密钥，用于AI智能评分和自动出题功能
- `ZHIPUAI_BASE_URL`: 智谱AI API的基础URL
- `ZHIPUAI_MODEL`: 使用的AI模型，推荐使用免费模型 `glm-4-flash`

**获取智谱AI API密钥:**
1. 访问 https://open.bigmodel.cn/
2. 注册账号并登录
3. 在控制台中创建API密钥
4. 将密钥配置到环境变量中

### Redis配置 (Redis Configuration) - 可选

```env
# Redis连接URL (可选，用于缓存和会话存储)
REDIS_URL=redis://localhost:6379/0
```

**配置说明:**
- `REDIS_URL`: Redis连接URL，用于缓存和会话存储
- 如果不使用Redis，可以不配置此变量

### 邮件配置 (Email Configuration) - 可选

```env
# 邮件服务器配置 (可选，用于发送通知邮件)
EMAIL_HOST=smtp.gmail.com
EMAIL_PORT=587
EMAIL_USE_TLS=True
EMAIL_HOST_USER=your-email@gmail.com
EMAIL_HOST_PASSWORD=your-email-password
```

**配置说明:**
- `EMAIL_HOST`: SMTP邮件服务器地址
- `EMAIL_PORT`: SMTP端口，通常为587或465
- `EMAIL_USE_TLS`: 是否使用TLS加密
- `EMAIL_HOST_USER`: 发送邮件的邮箱地址
- `EMAIL_HOST_PASSWORD`: 邮箱密码或应用专用密码

### 安全设置 (Security Settings)

```env
# CORS允许的源 (跨域请求配置)
CORS_ALLOWED_ORIGINS=http://localhost:8080,http://127.0.0.1:8080

# CSRF信任的源 (CSRF保护配置)
CSRF_TRUSTED_ORIGINS=http://localhost:8080,http://127.0.0.1:8080
```

**配置说明:**
- `CORS_ALLOWED_ORIGINS`: 允许跨域请求的源地址
- `CSRF_TRUSTED_ORIGINS`: CSRF保护信任的源地址
- 生产环境需要添加实际的域名

### 静态文件配置 (Static Files Configuration)

```env
# 静态文件URL
STATIC_URL=/static/

# 媒体文件URL
MEDIA_URL=/media/

# 静态文件根目录
STATIC_ROOT=/app/static

# 媒体文件根目录
MEDIA_ROOT=/app/media
```

**配置说明:**
- `STATIC_URL`: 静态文件的URL前缀
- `MEDIA_URL`: 媒体文件的URL前缀
- `STATIC_ROOT`: 静态文件收集的根目录
- `MEDIA_ROOT`: 媒体文件存储的根目录

## 🚀 快速配置指南

### 1. 复制配置模板
```bash
# 复制环境配置模板
cp env.example .env
```

### 2. 编辑配置文件
使用文本编辑器打开 `.env` 文件，修改以下关键配置：

```env
# 必须修改的配置
SECRET_KEY=your-random-secret-key-here
DB_PASSWORD=your-mysql-password

# 可选配置 (如果需要AI功能)
ZHIPUAI_API_KEY=your-actual-api-key
```

### 3. 生成安全密钥
```bash
# 使用Python生成随机密钥
python -c "from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())"
```

### 4. 验证配置
```bash
# 进入后端目录
cd server

# 激活虚拟环境
source .venv/bin/activate  # Linux/Mac
# 或
.venv\Scripts\activate     # Windows

# 测试配置
python manage.py check
```

## 🛠️ 常见配置问题

### 1. 数据库连接失败
**问题**: 无法连接到MySQL数据库
**解决方案**:
```env
# 检查数据库配置
DB_HOST=localhost          # 确保MySQL服务运行
DB_PORT=3306              # 确保端口正确
DB_USER=root              # 确保用户名正确
DB_PASSWORD=your_password # 确保密码正确
DB_NAME=db_exam           # 确保数据库存在
```

### 2. SECRET_KEY未设置
**问题**: Django启动时提示SECRET_KEY未设置
**解决方案**:
```env
# 设置一个安全的SECRET_KEY
SECRET_KEY=django-insecure-your-very-long-random-string-here
```

### 3. AI功能无法使用
**问题**: AI智能评分和自动出题功能无法使用
**解决方案**:
```env
# 配置智谱AI API密钥
ZHIPUAI_API_KEY=your-actual-api-key-from-zhipuai
```

### 4. 跨域问题
**问题**: 前端无法访问后端API
**解决方案**:
```env
# 配置CORS允许的源
CORS_ALLOWED_ORIGINS=http://localhost:8080,http://127.0.0.1:8080
CSRF_TRUSTED_ORIGINS=http://localhost:8080,http://127.0.0.1:8080
```

## 📝 配置最佳实践

### 1. 安全性
- **永远不要**将 `.env` 文件提交到版本控制系统
- 使用强密码和随机生成的SECRET_KEY
- 生产环境设置 `DEBUG=False`

### 2. 环境分离
- 开发环境: `DEBUG=True`，使用本地数据库
- 测试环境: 使用测试数据库
- 生产环境: `DEBUG=False`，使用生产数据库

### 3. 配置管理
- 使用 `env.example` 作为配置模板
- 为不同环境创建不同的配置文件
- 定期更新和检查配置
