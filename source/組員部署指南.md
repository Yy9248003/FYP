### 方法2: 手動部署

## 📝 手動部署步驟

### 步驟1: 獲取項目代碼
```bash
# 克隆項目
git clone https://github.com/HUANHJiali/25FYP.git
cd 25FYP/source
```

### 步驟2: 數據庫設置
```bash
# 1. 啟動 MySQL 服務
# Windows: 通過服務管理器啟動 MySQL
# Linux: sudo service mysql start
# Mac: brew services start mysql

# 2. 創建數據庫
mysql -u root -p
```
```sql
CREATE DATABASE db_exam CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
EXIT;
```

```bash
# 3. 導入數據庫
mysql -u root -p db_exam < database/db_exam.sql
mysql -u root -p db_exam < database/init_practice_data.sql
```

### 步驟3: 後端設置
```bash
# 1. 進入後端目錄
cd server

# 2. 創建虛擬環境
python -m venv .venv

# 3. 激活虛擬環境
# Windows:
.venv\Scripts\activate
# Linux/Mac:
source .venv/bin/activate

# 4. 安裝依賴
pip install -r requirements.txt

# 5. 運行數據庫遷移
python manage.py migrate

# 6. 創建管理員賬戶
python manage.py createsuperuser
# 用戶名: admin
# 密碼: 123456

# 7. 啟動後端服務
python manage.py runserver
```

### 步驟4: 前端設置 (新終端)
```bash
# 1. 進入前端目錄
cd client

# 2. 安裝依賴
npm install

# 3. 啟動前端服務
npm run serve
```

## 🛠️ 常見問題解決

### 問題1: 端口被占用
```bash
# 查看端口使用情況
netstat -ano | findstr :8000  # Windows
lsof -i :8000                 # Linux/Mac

# 殺死占用進程
taskkill /PID <進程ID> /F     # Windows
kill -9 <進程ID>              # Linux/Mac
```

### 問題2: 數據庫連接失敗
```bash
# 檢查 MySQL 服務狀態
# Windows: 服務管理器查看 MySQL 服務
# Linux: sudo service mysql status
# Mac: brew services list | grep mysql

# 測試數據庫連接
mysql -u root -p -e "SHOW DATABASES;"
```

### 問題3: 依賴安裝失敗
```bash
# Python 依賴
pip install --upgrade pip
pip install -r requirements.txt

# Node.js 依賴
npm cache clean --force
npm install
```

### 問題4: 權限問題 (Linux/Mac)
```bash
# 給腳本添加執行權限
chmod +x setup_env.sh
chmod +x setup_env.py
```
