# 🚀 智能考试系统 - 完整运行指南

## 📋 项目概述
- **项目名称**: 智能学生考试系统 (FYPPPP)
- **技术栈**: Django 4.1.3 + Vue.js 2.x + MySQL + 智谱AI
- **核心功能**: 练习试卷、AI智能评分、自动出题、用户管理、错题本

## 🖥️ 本地开发运行

### 前置要求
- ✅ **Python 3.9+** - 后端开发环境
- ✅ **Node.js 16+** - 前端开发环境
- ✅ **MySQL 8.0+** - 数据库服务
- ✅ **Git** - 版本控制工具

### 步骤1: 获取项目代码
```bash
# 克隆项目
git clone https://github.com/HUANHJiali/FYPPPP.git
cd FYPPPP/source

# 或者直接下载项目压缩包并解压
```

### 步骤2: 数据库设置
```bash
# 启动MySQL服务
# Windows: 通过服务管理器启动MySQL服务
# Linux: sudo service mysql start
# Mac: brew services start mysql

# 创建数据库
mysql -u root -p
CREATE DATABASE db_exam CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
exit

# 导入数据库结构和初始数据
mysql -u root -p db_exam < database/db_exam.sql
mysql -u root -p db_exam < database/init_practice_data.sql
```

### 步骤3: 后端环境配置
```bash
# 进入后端目录
cd server

# 创建Python虚拟环境
python -m venv .venv

# 激活虚拟环境
# Windows:
.venv\Scripts\activate
# Linux/Mac:
source .venv/bin/activate

# 升级pip
pip install --upgrade pip

# 安装Python依赖
pip install -r requirements.txt

# 返回项目根目录
cd ..
```

### 步骤4: 环境配置文件
```bash
# 复制环境配置模板
cp env.example .env

# 编辑.env文件，配置数据库连接信息
# 使用文本编辑器打开.env文件，修改以下配置：
```

**环境配置示例 (.env文件):**
```env
# 数据库配置
DB_NAME=db_exam
DB_USER=root
DB_PASSWORD=your_mysql_password
DB_HOST=localhost
DB_PORT=3306

# Django配置
SECRET_KEY=your-secret-key-here
DEBUG=True
ALLOWED_HOSTS=localhost,127.0.0.1,0.0.0.0

# AI配置 (可选，用于AI功能)
ZHIPUAI_API_KEY=your-zhipuai-api-key
ZHIPUAI_BASE_URL=https://open.bigmodel.cn/api/paas/v4
ZHIPUAI_MODEL=glm-4-flash
```

### 步骤5: 后端服务启动
```bash
# 进入后端目录
cd server

# 激活虚拟环境 (如果未激活)
# Windows: .venv\Scripts\activate
# Linux/Mac: source .venv/bin/activate

# 运行数据库迁移
python manage.py migrate

# 创建管理员账户
python manage.py createsuperuser
# 按提示输入用户名、邮箱和密码

# 启动Django开发服务器
python manage.py runserver
```

### 步骤6: 前端环境配置
```bash
# 新开一个终端窗口，进入前端目录
cd client

# 安装Node.js依赖
npm install

# 如果安装失败，尝试清理缓存后重新安装
npm cache clean --force
npm install
```

### 步骤7: 前端服务启动
```bash
# 在前端目录中启动开发服务器
npm run serve

# 或者构建生产版本
npm run build
```

### 访问地址
- **前端界面**: http://localhost:8080
- **后端API**: http://localhost:8000
- **管理后台**: http://localhost:8000/admin

## 🏭 生产部署（一键脚本）

适用场景：Ubuntu 20.04/22.04+ 或 Windows Server，一次性完成后端虚拟环境、依赖安装、数据库导入/迁移、静态文件收集、前端构建、Nginx 与 systemd 配置。

### Linux（Ubuntu）

```bash
sudo bash deploy/setup_server.sh \
  --project-root /path/to/FYP2025-main \
  --domain your.domain.com \
  --env-file /etc/exam/.env \
  --db-import yes

# 可选：签发 HTTPS 证书
sudo apt-get install -y certbot python3-certbot-nginx
sudo certbot --nginx -d your.domain.com
```

关键点：
- `.env` 推荐放在 `/etc/exam/.env`，并由 systemd `EnvironmentFile` 载入
- Nginx 样例位于 `deploy/nginx.conf`（包含 gzip、超时、/api 代理）
- systemd 样例位于 `deploy/exam-gunicorn.service`（含访问/错误日志与 graceful-timeout）

### Windows（PowerShell 以管理员运行）

```powershell
powershell -ExecutionPolicy Bypass -File deploy\setup_server.ps1 `
  -ProjectRoot "C:\\FYP2025-main" `
  -EnvFile "C:\\exam\\.env" `
  -DBImport Yes

# 手动运行（调试）
cd C:\\FYP2025-main\\source\\server
.\.venv\\Scripts\\Activate.ps1
python -m uvicorn server.asgi:application --host 0.0.0.0 --port 8000 --workers 2
```

驱动兼容：Windows 上如 `mysqlclient` 安装失败，脚本会自动回退 `PyMySQL`。

前端生产：`npm run build` 生成 `source/client/dist`，由 Nginx 直接提供并代理 `/api/`。

### 常见问题2: Python依赖安装失败
```bash
# 升级pip
pip install --upgrade pip

# 清理pip缓存
pip cache purge

# 重新安装依赖
pip install -r requirements.txt

# 如果仍有问题，尝试使用国内镜像
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple/
```

### 常见问题3: Node.js依赖安装失败
```bash
# 清理npm缓存
npm cache clean --force

# 删除node_modules目录
rm -rf node_modules
# Windows: rmdir /s node_modules

# 重新安装
npm install

# 如果仍有问题，尝试使用国内镜像
npm install --registry https://registry.npmmirror.com
```

### 常见问题4: 端口被占用
```bash
# 查看端口使用情况
# Windows:
netstat -ano | findstr :8000
netstat -ano | findstr :8080

# Linux/Mac:
lsof -i :8000
lsof -i :8080

# 杀死占用端口的进程
# Windows: taskkill /PID <进程ID> /F
# Linux/Mac: kill -9 <进程ID>
```

### 常见问题5: 前端权限问题 (Linux/Mac)
```bash
# 给node_modules/.bin目录添加执行权限
chmod +x node_modules/.bin/*

# 或者重新安装依赖
rm -rf node_modules package-lock.json
npm install
```

## 🎯 开发工作流程

### 日常开发
1. **启动后端服务**: `cd server && python manage.py runserver`
2. **启动前端服务**: `cd client && npm run serve`
3. **访问系统**: http://localhost:8080

### 代码修改
1. **后端修改**: 修改Django代码后，服务会自动重载
2. **前端修改**: 修改Vue代码后，浏览器会自动刷新
3. **数据库修改**: 修改模型后需要运行迁移

### 数据库迁移
```bash
# 创建迁移文件
python manage.py makemigrations

# 应用迁移
python manage.py migrate

# 查看迁移状态
python manage.py showmigrations
```
